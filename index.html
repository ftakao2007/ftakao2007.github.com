Fluentd

Apr 8th, 2013

何故、fluentdなのか？

概要

  • それぞれのサーバからログを収集し集約する為のアプリケーション
  • 全てのログをJSON形式で扱う
  • ログを見るのに直接サーバにログインしなくても良い
  • 複数のサーバのログが一箇所に集約される
    → AutoScalingのサーバの台数増減に対応できる

Munin

Apr 8th, 2013

ちょっとMuninいれてみました。

特徴

  • リソース監視
    → クリティカルな問題ではないが、何か重い時のボトルネック発見

長所、短所

長所

  • 手軽
  • yumで入る
  • DBが必要なく、httpdが動いていればOK

短所

  • 設定はCUIのみ
  • ユーザー管理の概念が無い
  • サーバクライアント方式で、被監視サーバにはクライアント(munin-node)を入れる必要がある

Gitメモ

Apr 7th, 2013

メモ書き

コンセプトは、曖昧に覚えていることをとりあえず書き出し、修正しながら身につける。

リモートリポジトリ

  • gitサーバ側に作成されるリポジトリ。クライアント側から作成することはできない。
  • このリポジトリが無いとpushできない

master

  • masterのブランチ
  • git commit -m "hoge" のように、commitを初めてした時に自動で作成される
  • ブランチは git branch で確認可能

origin

  • URLのエイリアスみたいなもの
  • git remote add origin https://github.com/ftakao2007/test.git で後ろのURLが登録される
